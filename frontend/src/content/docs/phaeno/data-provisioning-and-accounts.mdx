# Data provisioning

Phaeno controls organization access and the lifecycle of Phaeno-owned source samples and curated package versions. Customer and Partner operational data is a separate access class and is never automatically reused for curated packages.

Use the far-left sidebar beneath the toolbar to move among **Source registry**, **Curated catalog**, **Organization grants**, and **Governance**. The sidebar header identifies Data provisioning and, on wider screens, provides the pin or unpin icon. Pointer users can preview an unpinned sidebar by approaching the left edge; keyboard and touch users open it from the edge tab. Choosing a section closes an unpinned sidebar.

## Understand the lifecycle

1. Register Phaeno-owned source material and its ownership and de-identification evidence.
2. Complete the editable source revision and mark it ready.
3. Snapshot one exact ready revision into a curated draft.
4. Validate and publish an immutable package version.
5. Mark the version eligible for the intended external audience when appropriate.
6. Explicitly grant one exact version to one organization.
7. Preserve publication, grant, download, revocation, retirement, and governance history.

Each step is explicit. Source readiness does not publish a package. Publication does not grant access. Eligibility does not grant access. A later version does not upgrade an existing grant.

## Use the correct access

Only authorized Phaeno users register sources, publish versions, manage eligibility, create or upgrade grants, revoke access, or perform governance actions. Prospect, Customer, and Partner users can view only the exact active versions granted to their current organization.

Organization membership and lifecycle are managed in **Organizations**. A Prospect-to-Customer or Prospect-to-Partner conversion preserves the organization identity, memberships, grants, pinned versions, and audit history. Conversion never creates, replaces, upgrades, or revokes a curated-data grant.

## Protect the boundary

Operational Customer or Partner files do not enter the source pool automatically. Every curated package is derived from an approved Phaeno-owned source revision. Keep storage keys, internal evidence, another organization’s grant state, and governance investigation detail out of tenant-facing content.
