# Scientific approval and release readiness

Scientific approval is the final Lab Ops decision. It confirms that the authorized laboratory work satisfied the configured execution and exception gates. It is not a Customer file publication action.

## Enter scientific review

Move work into **Scientific review** only after required receipt, accession, executions, QC, libraries, batches, and sendout facts are complete for the approved release scope. Confirm that every assigned execution is completed or explicitly abandoned and that every blocking exception is resolved.

The Scientific Reviewer should be able to trace the submitted specimen through containers, material lots, equipment, protocol versions, captured results, library preparation, batches, and provider custody as applicable.

## Record the approval

Record the release-definition key and version, reviewer identity, approval time, and the permitted QC projection. The release definition establishes what Commercial may expect; it does not attach or generate files.

If the gate is not satisfied, leave the work in review and create the appropriate exception or rework action. Do not approve merely to move the Commercial order forward.

## Publish readiness

Approval moves the Lab work order to **Ready for release** and publishes the controlled Customer-safe milestone to Commercial. Commercial may receive the milestone, schedule health, expected timing, and safe QC projection defined by the contract.

## Respect the release boundary

Ready for release does not create, attach, upload, scan, or publish a result artifact. Order Ops must still verify the result manifest, checksums, malware scan, invoice synchronization, payment or approved credit, and publication decision.

If the Lab-to-Commercial event fails, retry the existing durable projection. Do not duplicate the approval, recreate the work order, or manually mark the Customer file released.
