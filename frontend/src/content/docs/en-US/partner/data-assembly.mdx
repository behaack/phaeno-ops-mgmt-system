# Request data assembly

Partner organization administrators submit files and metadata under an active Phaeno assembly profile, receive a per-job quote, and retrieve outputs for delivery through their own customer relationships.

## Create an input revision

1. Open **Data assembly** and create a draft.
2. Enter the Partner project or reference name.
3. Choose an active assembly profile available to the selected Partner.
4. Provide every required metadata value and data declaration.
5. Upload allowed file types within the stated size limits.

Files must complete the managed upload and clean-scan checks before submission. The current workflow does not authorize protected health information.

## Submit and correct

Submission freezes an immutable input revision. Phaeno may accept it for quoting, request field- or file-specific changes, or reject it with a Partner-visible reason.

If changes are requested, create a corrected revision and resubmit. Earlier revisions remain preserved for traceability.

## Accept the job quote

Assembly work is priced per job. Phaeno issues an itemized quote that becomes visible after its QuickBooks Online estimate synchronizes successfully. A Partner administrator accepts the current, unexpired quote and supplies the required purchase-order number.

An accepted quote freezes the validated input, profile, and commercial snapshot. Later scope changes use a separate immutable change quote.

## Track and download outputs

The request shows intake review, queued work, processing, output review, output availability, holds, and completion. An approved output release contains an immutable manifest and one or more checksummed files tied to the source input revision and assembly profile.

Credit-approved Partners use Net 30 and may download scientifically ready outputs after the completion invoice synchronizes. Partners without approved assembly credit can see that outputs are ready but cannot download them until QuickBooks confirms payment.

A corrected output is a new release. Partner users download the release and deliver it through their own customer process; Phaeno Portal does not collect the Partner’s end-customer identity or deliver directly to that customer.

Before quote acceptance, an administrator may withdraw the request. After acceptance, use the cancellation-request workflow so completed work and financial history remain preserved.
