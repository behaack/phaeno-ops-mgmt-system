# Data governance and downloads

Phaeno may revoke a grant, quarantine a package version, or retire a package as scientific, contractual, or governance needs change. These controls stop future portal access but cannot recall files that were already downloaded.

## Respond to a package notice

When Phaeno sends a package-governance instruction:

1. Confirm the package name and exact version.
2. Stop new use or distribution when the notice requires it.
3. Follow the safe action stated by Phaeno for copies already held by your organization.
4. If an attestation is requested, an organization administrator should record the organization’s response in the portal.

Do not include downloaded data, confidential scientific content, credentials, or internal investigation details in an attestation or support message.

## Understand package states

| State or event | Effect on your organization |
| --- | --- |
| Grant revoked | Future access to the granted package version stops. Earlier downloads are not recalled. |
| Version quarantined | Future access is blocked while Phaeno investigates or resolves the issue. |
| Package retired | The package is no longer available for new use through the portal. |
| New version published | Existing grants do not move automatically; Phaeno must grant the new version separately. |

## Review download history

Prospect organization administrators can review download history for their organization. Use it to determine which package version and file were downloaded, by whom, and when. Download history does not prove that every local copy was deleted or that downstream use stopped; follow your organization’s governance process and any instruction from Phaeno.

Phaeno retains audit history for grants, downloads, and governance actions. Do not ask support to delete history to conceal an access or download event.
